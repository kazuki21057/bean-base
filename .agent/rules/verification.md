# 検証を意識したコーディング
- Antigravityが`flutter run`を実行したとき、アプリの挙動がわかるようログを出力するようにして。

# 変更後の検証方法
- 必ず`flatter test`を実行し、すべてパスすることを確認して。
- 必ず`flutter run`を実行し、動作が正常であることを確認して。
- **マスターのUIや機能を追加・修正する際は、Beanだけでなく、Filter、Grinder、Dripperなど、すべての関連するタブ・フォームに対して漏れなく実装すること。**
- **ブラウザサブエージェント等を用いて、動作（例: 画像アップロードボタンがクリック可能かなど）を視覚的にも検証すること。**

# コマンド制御
コマンド「\\end」がプロンプト中にあれば下記を実行する。
- 次回何をすればいいのかをまとめる。
- これまでの作業をgit pushする
- ユーザに指摘されたことを再度指摘されないようルールに追加する
- タスクや実装計画を作成する際、これまでのCycle数（例: Cycle 16, 17など）を正しく把握し、docsディレクトリ配下等で連番管理を間違えないようにする。

コマンド「\\start」がプロンプト中にあれば下記を実行する。
- git pullする
- やることを整理し、やることを提示する
