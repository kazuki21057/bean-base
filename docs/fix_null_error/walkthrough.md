# 修正内容の確認 (Walkthrough) - キーマッピング対応

## 実施した変更 (キーマッピング対応)
ログ解析の結果、Google Sheets の列名が日本語であり、アプリ側の英語フィールド名と一致していないことが "Unknown" やデータ欠損の根本原因でした。また、シート名の間違いもありました。これらを修正しました。

### 1. キーマッピングの実装 (`sheets_service.dart`)
Service層で、日本語のキーをモデルが期待する英語のキーに変換する `_remapKeys` メソッドを実装し、全データ取得メソッドに適用しました。

例:
- `記録ID` -> `id`
- `メソッド名` -> `name`
- `豆の量(g)` -> `beanWeight`
- など、全項目のマッピングを定義済みです。

### 2. シート名の修正
Grinder (ミル) のシート名が誤っていました。
- 誤: `grinder_master`
- 正: `mill_master`

### 3. その他
これまでの「Null除去」「型ガード」などの堅牢化策はそのまま維持しています。

## 検証方法
1. アプリを再起動 (`R` または `flutter run`) してください。
2. Dashboard にスコアが表示されていること。
3. Logs に詳細情報が表示され、クリック可能であること。
4. Masters の各リストが "Unknown" から正しい名前に変わっていること。
を確認してください。
