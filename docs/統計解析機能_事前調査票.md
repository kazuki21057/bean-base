# 統計解析機能 設計書作成のための事前調査票

## このファイルの目的

このリポジトリ(ドリップコーヒー抽出履歴管理アプリ)に統計解析・予測機能を追加するための**設計書を別途作成する**。
設計書を実装者(Claude Code / Sonnet)が推論の余地なく実装できる精度で記述するには、既存コードベースの正確な情報が必要である。

**あなた(Claude Code)への指示**: リポジトリを調査し、下記の質問すべてに回答せよ。回答は本ファイル末尾の「回答欄」に追記する形式とし、推測ではなく実コードの該当箇所(ファイルパス・行番号)を根拠として示すこと。コードの変更は一切行わないこと。

---

## 背景: 実装予定の機能一覧

| ID | 機能 | 分類 |
|----|------|------|
| F1 | 重回帰分析(産地=国別ダミー変数、目的変数=総合評価、交互作用項含む)。VIF・調整済みR²・AICによる診断付き | 既存データの解析(振り返り) |
| F2 | 味覚5軸(甘味・酸味・苦味・複雑さ・フレーバー)のPCA。既存のPCA実装を拡張し、主成分の深掘り解釈を出力 | 既存データの解析(拡張) |
| F3 | レシピ提案「今日はこのレシピはいかが?」+ 提案履歴の記録(将来のA/Bテスト用データ蓄積) | 予測系の布石 |
| F4 | 属性ベースレシピ推薦(国×焙煎度 → 最適湯温・挽き目)。発展形としてガウス過程回帰+ベイズ最適化(獲得関数EI)をJSローカル実装 | 予測 |
| F5 | 豆の好みプロファイル(例:「エチオピア浅煎りを高評価する傾向」)。途中統計量の表示、定期更新、更新履歴の保存 | 予測 |
| F6 | 記録項目の追加候補: 焙煎日(または購入日)、抽出日、蒸らし時間、総抽出時間、brew ratio自動計算 | データ基盤 |

### 設計方針(確定事項)

- 数値計算(回帰・PCA・ガウス過程・ベイズ更新)はすべて**JSローカル計算**とする。LLMには計算させない(再現性・正確性の担保のため)
- Gemini API の役割は**計算結果の自然言語解釈のみ**に限定する
- データ件数は現在約140件。目的変数は総合評価を第一とする
- 産地はまず国単位で集約する(データ増加後に細分化の可能性あり)

### 【既存実装】

**Q10. 既存PCA実装: `lib/services/statistics_service.dart`(356行)。行列演算は `ml_linalg` を使うが、固有値分解は自前のJacobi法**
- import: `statistics_service.dart:2` `package:ml_linalg/linalg.dart`、`:3` `dart:math`
- `calculatePca()`(140-212行目):
  - 3件未満のデータでは実行不可(141行目)
  - 特徴量は味覚6軸(香り・酸味・苦味・甘味・複雑さ・フレーバー、145行目)。**総合評価(scoreOverall)はPCA入力から除外**されている
  - `ml_linalg` の `Matrix.fromList` で行列化(155行目)、列平均を引いて中心化(158-159行目)
  - 共分散行列 `C = (Xᵀ·X)/(n-1)` を `ml_linalg` の転置・乗算演算子で計算(161-164行目)。`ml_linalg` は基本線形代数(行列積・転置・平均)にのみ使用され、固有値分解機能は使っていない
  - 固有値分解は自前実装の `_jacobiEigenvalueAlgorithm()`(215-334行目) — 古典的な巡回式Jacobi回転法。生の `List<List<double>>` で実装(`maxIter: 100`, `tol: 1e-10`、215行目)。240-277行目のコメントには回転角の式についての試行錯誤の跡があり、検証済みライブラリではなく手製・未十分検証の実装であることが読み取れる
  - 固有値降順で上位2成分(PC1/PC2)を採用(328行目)し、各特徴量の寄与度(負荷量)算出とデータの射影を実施(187-204行目)し `PcaPoint`/`PcaComponent` を生成
  - エラー時は空の `PcaResult([], [])` を返しつつ `print` でログ出力するのみ(143, 207-211行目)
- 同ファイル内に `calculateKPI()`(91-99行目、総抽出回数・総豆消費量・平均総合評価)、`calculateRadarData()`(101-137行目、属性別平均・比較対象との差分)もあるが、いずれも単純な算術でありPCA以外の高度な統計処理はなし
- **回帰分析・ガウス過程のコードはリポジトリ内に一切存在しない**(`lib/` 全体を検索して `statistics_service.dart` のみが統計/ML関連ファイル)
