# サイクル15: ビジュアル改善とデータ管理強化

## 変更内容
1. **画像表示の統一 (BeanImage)**
   - ネットワーク画像 (`http`) とローカル画像 (ファイルパス) の両方に対応する `BeanImage` ウィジェットを作成しました。
   - Web版ではローカルパスの代わりにプレースホルダーを表示するように調整しました（条件付きインポートを使用）。
   - `MasterListScreen`, `MasterDetailScreen`, `HomeScreen`, `CalculatorScreen` 全体で `Image.network` を `BeanImage` に置き換え、表示崩れを防ぎました。

2. **ヒーローアニメーション (Hero Animation)**
   - リスト画面から詳細画面への遷移時に、画像が滑らかに拡大縮小するアニメーションを追加しました。

3. **Method (レシピ) 編集機能**
   - 既存メソッドの編集時に、保存されているステップ (`PouringStep`) を読み込む処理を追加しました。
   - ステップの追加・更新・削除における同期ロジックを実装しました。

4. **Empty State (空の状態) の改善**
   - ログや豆ストックが空の場合に、ユーザーに次のアクション（豆の追加や抽出）を促すリッチな表示を追加しました。

## 検証
- **コード解析**: `flutter analyze` を実行し、主要なエラーがないことを確認。
- **クロスプラットフォーム対応**: `BeanImage` の内部で `dart:io` (Mobile/Desktop) と `dart:html` 相当 (Web) の処理を分岐させ、Webビルドでのクラッシュを回避しました。
